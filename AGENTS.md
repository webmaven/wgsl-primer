# Tour of WGSL - Agent Instructions

## Overview

This repository contains the source code for the Tour of WGSL (WebGPU Shading Language) website. The site is generated using **ProperDocs** (with the MaterialX theme), and the interactive visualizers are powered by TypeScript and compiled/bundled using **Vite**.

> [!IMPORTANT]
> **Virtual Environment Mandatory**: All Python-based commands, Python-based test scripts (such as Playwright audit scripts), and `properdocs` invocations MUST be run using the project's local virtual environment (`venv`). A virtual environment must be created at the `venv/` directory under the project root so npm scripts (e.g. `npm run serve`, `npm run build`) can resolve `venv/bin/properdocs` correctly.

## Technology Stack

- **Static Site Generator**: ProperDocs (Python-based)
- **Theme & Formatting**: ProperDocs MaterialX Theme
- **Mathematical Rendering**: MathJax v3 & `pymdownx.arithmatex` (enables LaTeX math with `\(...\)` and `\[...\]` delimiters)
- **Interactive Visualizers**: WebGPU visualizers written in TypeScript, using **CodeMirror 6** (with `@iizukak/codemirror-lang-wgsl`) for live editing
- **Assets & Bundling**: Vite (compiles TypeScript and SCSS/CSS into compiled assets in `docs/assets/`)
- **Scripting**: TypeScript (ESLint & Prettier)
- **CI/CD**: GitHub Actions (using modern checkout@v4, setup-node@v4, setup-python@v5, configure-pages@v5 to avoid deprecated Node environments) and GitHub Pages deployment

## Directory Structure

- `docs/`: Contains the Markdown files for the tutorial sections and lessons, plus the compiled JS and CSS assets. Note: Do NOT manually modify files in `docs/assets/` directly, as they are generated by the Vite build process.
- `assets/ts/`: Contains the TypeScript source code for interactive WGSL visualizers and the site's frontend logic.
- `hooks/`: Contains custom Python hooks used by ProperDocs (e.g., `wgsl_tour.py`) to process pages or customize layout.
- `properdocs.yml`: The main ProperDocs configuration file.
- `vite.config.ts`: Configuration file for the Vite asset compiler/bundler.
- `.github/workflows/`: Contains the GitHub Actions workflows for continuous integration, validation, and deployment.

## Development Workflow

- **Install Dependencies**:
  - Set up a Python virtual environment at the `venv` directory: `python3 -m venv venv`.
  - Activate the virtual environment: `source venv/bin/activate`.
  - Install Python dependencies (including ProperDocs) within the activated environment: `pip install -r requirements.txt`.
  - Install Node.js dependencies: `npm install`.
- **Run Locally**:
  - Run the development server with hot reloading for both TypeScript (Vite) and Markdown (ProperDocs): `npm run serve` (which starts `properdocs serve` using `venv/bin/properdocs serve` and Vite compilation).
- **Linting & Type Checking**:
  - Run `npm run lint` to lint the TypeScript files.
  - Run `npx tsc --noEmit` to verify type safety.
- **Formatting**:
  - Run `npm run format` to run Prettier across the codebase.
- **Build**:
  - Running `npm run build` will compile the TypeScript assets to `docs/assets/` and generate the final static site in the `site/` directory using `properdocs build` (resolved via `venv/bin/properdocs`).

## Publishing Process

The site uses automated GitHub Actions workflows for deployment:

1. **Production**: Merges or pushes to the `main` branch trigger the `deploy-main.yml` workflow, which installs Python and Node dependencies, builds the static site to the `site/` directory, and deploys it to the `gh-pages` branch for hosting on GitHub Pages.
2. **Pull Requests**: For PRs, the `build-pr.yml` and `deploy-pr.yml` workflows build a preview of the site and deploy it to a `pr-preview/pr-<NUMBER>` subfolder on the `gh-pages` branch.

## Agent Instructions

1. **TypeScript Edits**: Always run `npm run lint` and `npx tsc --noEmit` after modifying TypeScript code to verify correctness.
2. **Read-Only Output**: Do NOT manually modify the `site/` or `docs/assets/` directories, as these are generated artifacts.
3. **Environment Constraints**: Do not place Git repositories (`.git/`) or sensitive configurations inside the `docs/` directory as it is publicly served.
4. **Vite Compiles to Docs**: When working on visualizers (`assets/ts/`), remember that Vite transpiles these into the `docs/assets/` folder, which ProperDocs then serves. Always test using `npm run serve` or compile using `npm run build`.
5. **Inline Notation & Math Style Guide**: When editing or writing markdown content in the `docs/` directory, strictly adhere to the [Inline Notation & Math Style Guide](STYLE_GUIDE.md) to maintain zero baseline shifts and layout consistency:
   - Use inline code backticks `` `code` `` **strictly** for concrete, actual, or non-parameterized code.
   - Use HTML `<code>` and colored `<span>` template classes for generic signatures/syntax templates (e.g. `<code>array&lt;<span class="template template-array-t">T</span>&gt;</code>`).
   - Use Standalone Template Chips (`<span class="template">`) outside signatures only when describing parameters.
   - Use inline math `\( ... \)` **strictly** for mathematical formulas, algebra, and geometric coordinates or indices. Single dollar signs are prohibited.
6. **Virtual Environment Usage**: When invoking Python scripts, running audit tools, or executing `properdocs` commands, always utilize the local Python virtual environment (`venv/bin/python` or `venv/bin/properdocs`) or activate the virtual environment (`source venv/bin/activate`) to ensure the correct package versions (including ProperDocs and Markdown extensions) are resolved. Do not install or run packages globally.
